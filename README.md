# Лексический анализатор (Вариант 8)

Веб-приложение для лексического анализа логических выражений на Go + HTMX.

## Описание

Лексический анализатор для распознавания логических выражений согласно варианту 8:
- Идентификаторы (начинаются с буквы, могут содержать буквы, цифры и _)
- Константы: `0` и `1`
- Присваивание: `:=`
- Логические операции: `or`, `xor`, `and`, `not`
- Круглые скобки: `(` `)`
- Разделитель выражений: `;`
- Комментарии: `//` (однострочные) и `/* */` (многострочные)

## Возможности

- ✅ Распознавание всех типов лексем
- ✅ Определение позиции лексем (строка и столбец)
- ✅ Обработка комментариев
- ✅ Детальный вывод ошибок
- ✅ Статистика анализа
- ✅ Интерактивный веб-интерфейс с HTMX
- ✅ Адаптивный дизайн


### Шаг 1: Подготовка репозитория

### Шаг 2: Настройка на Render.com

1. Зарегистрируйтесь на [render.com](https://render.com)
2. Нажмите **"New +"** → **"Web Service"**
3. Подключите ваш GitHub репозиторий
4. Настройте параметры:

**Build & Deploy Settings:**
- **Name**: `lexical-analyzer` (или любое другое имя)
- **Environment**: `Go`
- **Build Command**: `go build -o app`
- **Start Command**: `./app`

**Advanced Settings:**
- **Auto-Deploy**: `Yes` (автоматический деплой при push)

5. Нажмите **"Create Web Service"**

### Шаг 3: Ожидание деплоя

Render автоматически:
- Установит Go
- Скачает зависимости из `go.mod`
- Соберет приложение
- Запустит сервер

Через 2-3 минуты ваше приложение будет доступно по ссылке вида:
`https://your-app-name.onrender.com`

## Примеры использования

### Пример 1: Простое выражение
```
a := 1;
b := 0;
result := a and b;
```

### Пример 2: Сложное выражение с комментариями
```
// Инициализация переменных
x := 1;
y := 0;

/* Вычисление логических операций */
res1 := x or y;
res2 := (x and y) xor not x;
```

### Пример 3: Ошибки
```
a = 1;        // Ошибка: нет :=
b := 2;       // Ошибка: 2 не является допустимой константой
c := a & b;   // Ошибка: & не является допустимой операцией
```

## Архитектура лексического анализатора

### Конечный автомат (FSM)

Лексический анализатор построен на основе конечного автомата:

**Состояния:**
- `AUTO_N` - начальное состояние
- `AUTO_ID` - чтение идентификатора
- `AUTO_NUM` - чтение константы (0 или 1)
- `AUTO_COLON` - обработка `:` (ожидание `=`)
- `AUTO_ER` - состояние ошибки
- `AUTO_S` - конечное состояние

**Переходы:**
- Буква → чтение идентификатора
- Цифра (0,1) → константа
- `:` → проверка на `:=`
- Символы `(`, `)`, `;` → соответствующие токены

### Распознаваемые токены

| Тип токена | Пример | Описание |
|------------|--------|----------|
| Идентификатор | `a`, `result`, `var_1` | Начинается с буквы |
| Константа | `0`, `1` | Логические константы |
| Присваивание | `:=` | Оператор присваивания |
| OR | `or` | Логическое ИЛИ |
| XOR | `xor` | Исключающее ИЛИ |
| AND | `and` | Логическое И |
| NOT | `not` | Логическое НЕ |
| Скобки | `(`, `)` | Группировка выражений |
| Разделитель | `;` | Конец выражения |

## Обработка ошибок

Анализатор обнаруживает следующие ошибки:
- Недопустимые символы
- Неправильное использование операторов
- Незакрытые комментарии
- Неверный формат констант (цифры кроме 0 и 1)
